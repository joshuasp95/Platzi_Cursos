# Implementación de Réplicas en Postgres con Docker

Este tutorial te guiará en el proceso de configuración de una replicación maestro-esclavo en PostgreSQL utilizando Docker. Necesitarás tener Docker instalado en tu sistema.

## Pre-requisitos

- Docker instalado.
- Conocimientos básicos de Docker y PostgreSQL.

## Paso 1: Descarga de la imagen de PostgreSQL

Primero, descarga la imagen de PostgreSQL utilizando el siguiente comando:

```sh
docker pull postgres:alpine
```

## Paso 2: Creación de los contenedores Docker

Crearás dos contenedores: uno para el servidor maestro y otro para el esclavo.

## Paso 3: Configuración de la Conexión en PgAdmin

Establecerás las conexiones a los servidores de base de datos en pgAdmin. Necesitarás proporcionar la información correcta para el servidor maestro y el servidor esclavo.

Ahora, configurarás la réplica. Primero, copia toda la información de la base de datos maestra en la réplica, y luego inicia la réplica en modo esclavo.

**Acceder al contenedor:**

**Borrar los datos actuales y copiar los datos del maestro:**

## Paso 6: Verificación de la Replicación

Para verificar que la replicación ha sido configurada correctamente, intenta crear una base de datos en la réplica y en el maestro.
